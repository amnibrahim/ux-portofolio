Coworking Space Map

Coworking Space

Problem

Today, there are many people that work in the tech industry. However, the limitation in finding space to be able to work in a team has made it difficult for people to communicate and discuss their project. In this project, a web based map application was made by making use of ReactJS router and a map API.

Role

Front end developer - Develop Map Application in ReactJS

Process

First, a list of coworking space in Jakarta was gathered in a json file. Each value contains data such as name, price, open hour, contact, description and coordinate location.

Because the there are many data that would be displayed for each individual coworking space, thus, ReactJS router was used. The purpose is to display a list of available coworking space where each item would open a child component that shows the price, open hour, desciption, etc. By enabling ReactJS router, everytime a component is rendered, the page wouldn't need to refresh the page.

Challange

During the development process, Google Maps API was considered to be the map component on the page. The decision was to take the advantage of "seacrh location by name" feature that Google Maps has which would make it easier to develop. However, by 2018, Google has decided to make this service to be no longer free. Although there is a $200 total of free requests, it is considered to be unsustainable. Once this free period ends, the page needs to use another free map API which obviously would reconstructure the whole component. Therefore, Mapbox API was used because it has complete documentation, different maps style and free. The only challange is that the json file that contains the raw data needs to be added with latitude and longitude coordinate as these would be used as input for icons on the maps.

Outcome

There are two three main components in the page. The navigation, the listed item, and the map. The direction in using this web app is from the left side which is the navigation component. In this component, user are able to decide which part of Jakarta that they are interesting to see a list of coworking spaces.

The second component would let user to choose one item from a list of coworking space filtered by region in Jakarta. When user finally click an item, a complete information of price, open hour, contact and description. Once this component comes up, it would also trigger the third component which is the map component. A set of icons would show up in the map which indicate the location of the specified coworking space.

Improvement

There are some things that could be improved, one of them is to provide an interactive icon on the map. Another thing is to provide a direction between user's location to selected icon on the map. In addition, currently, this project is unresponsive and was made for desktop only. The reason is because the map component requires 50% of screen width for user to be able to comfortably access the content which makes it difficult to be applied in mobile. However, this project would always be open to new ideas and would listen to any improvements.

Tools

React.js, SASS, HTML/CSS/JavaScript, Figma
